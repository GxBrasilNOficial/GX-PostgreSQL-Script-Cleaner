## 🇧🇷 Português (Brasil)

### O que é

Ferramenta web para corrigir scripts SQL gerados pelo **GeneXus** para execução no **PostgreSQL / pgAdmin 4**.

O GeneXus costuma gerar scripts com espaços não-quebráveis (`\u00A0`) e outros caracteres invisíveis que causam erros de sintaxe no PostgreSQL. Esta ferramenta remove esses problemas automaticamente.

### Problemas corrigidos

- Espaços não-quebráveis e caracteres Unicode invisíveis
- BOM (Byte Order Mark) no início do arquivo
- Quebras de linha inconsistentes (`\r\n` → `\n`)
- Espaços extras no final e no interior das linhas
- Linhas em branco excessivas
- Espaço indevido antes de vírgula e parêntese de fechamento
- Ausência de ponto e vírgula (`;`) no final das instruções, incluindo instruções multilinhas como `ALTER TABLE ... ADD COLUMN` e `ALTER TABLE ... ALTER COLUMN`

### Como usar

1. Acesse o app: **[https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html](https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html)**
2. Cole o script gerado pelo GeneXus no campo esquerdo
3. Clique em **▶ Limpar Script**
4. Copie o resultado e execute no pgAdmin 4

### Tecnologia

HTML + JavaScript puro. Roda 100% no navegador, sem servidor, sem consumo de tokens de IA.

## 🇺🇸 English

### What is it?

A web tool to fix SQL scripts generated by **GeneXus** for execution in **PostgreSQL / pgAdmin 4**.

GeneXus often generates scripts containing non-breaking spaces (`\u00A0`) and other invisible characters that cause syntax errors in PostgreSQL. This tool automatically removes those issues.

### Issues fixed

- Non-breaking spaces and invisible Unicode characters
- BOM (Byte Order Mark) at the start of the file
- Inconsistent line endings (`\r\n` → `\n`)
- Trailing and extra internal spaces
- Excessive blank lines
- Unwanted space before commas and closing parentheses
- Missing semicolons (`;`) at the end of statements, including multi-line statements such as `ALTER TABLE ... ADD COLUMN` and `ALTER TABLE ... ALTER COLUMN`

### How to use

1. Open the app: **[https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html](https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html)**
2. Paste the GeneXus-generated script into the left field
3. Click **▶ Limpar Script**
4. Copy the result and run it in pgAdmin 4

### Technology

Pure HTML + JavaScript. Runs 100% in the browser — no server, no AI token consumption.

## 🇯🇵 日本語

### これは何ですか？

**GeneXus** が生成した SQL スクリプトを **PostgreSQL / pgAdmin 4** で実行できるよう修正するウェブツールです。

GeneXus はノーブレークスペース（`\u00A0`）などの不可視文字を含むスクリプトを生成することがあり、PostgreSQL で構文エラーが発生する原因となります。このツールはそれらの問題を自動的に修正します。

### 修正される問題

- ノーブレークスペースおよびその他の不可視 Unicode 文字
- ファイル先頭の BOM（バイトオーダーマーク）
- 不統一な改行コード（`\r\n` → `\n`）
- 行末および行内の余分なスペース
- 連続する空白行
- カンマや閉じ括弧の前の不要なスペース
- 各ステートメント末尾のセミコロン（`;`）の欠落（`ALTER TABLE ... ADD COLUMN` や `ALTER TABLE ... ALTER COLUMN` などの複数行ステートメントを含む）

### 使い方

1. アプリを開く: **[https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html](https://gxbrasilnoficial.github.io/GX-PostgreSQL-Script-Cleaner/gx_postgresql_script_cleaner.html)**
2. 左側のフィールドに GeneXus が生成したスクリプトを貼り付ける
3. **▶ Limpar Script** をクリック
4. 結果をコピーして pgAdmin 4 で実行

### 技術情報

純粋な HTML + JavaScript で構築されています。サーバー不要、AI トークン消費なし、すべてブラウザ上で動作します。
